# 搜索词飞轮：膨胀 + AI 筛选闭环

## 一句话说明

用初始种子词搜索闲鱼 → 收集标题 → AI 从标题中提取候选词 → AI 筛选 → 优质词进入下一轮搜索 → 循环往复。

词库自我膨胀但被 AI 控制在"有价值"的范围内。

---

## 整体流程

```
┌─────────────────────────────────────────────────────────┐
│                    搜索词飞轮                            │
│                                                         │
│  种子词 ──→ 闲鱼搜索 ──→ 标题收集 ──→ AI提取候选词      │
│    ↑                                        │           │
│    │                                        ▼           │
│    │                               AI 三层筛选          │
│    │                          (去噪→去重→评分)          │
│    │                                        │           │
│    │                                        ▼           │
│    └─────────── 通过筛选的词 ←────── 候选词池           │
│                 进入下一轮搜索                           │
└─────────────────────────────────────────────────────────┘
```

## 第一步：标题收集

### 输入
一组搜索词，例如：
```
["自行车", "捷安特ATX830", "铝合金山地车"]
```

### 执行
用每个词在闲鱼搜索，收集搜索结果中每条商品的标题和 fishTags。

### 输出
标题列表（每个词搜 20 条，3 个词 = 60 条标题），例如：

```
- "捷安特ATX830铝合金27速山地车 油碟刹车 九成新"
- "全新捷安特ATX830 禧玛诺变速 26寸 山地自行车"
- "闲置自用美利达公爵600 搬家便宜出 不议价"
- "喜德盛传奇500 27.5寸 气压前叉 仅自提"
```

## 第二步：AI 提取候选词

### Prompt 模板

```
你是闲鱼选品关键词提取专家。以下是从闲鱼搜索"自行车"类目得到的商品标题列表。

请完成以下任务：

1. 【品牌词】提取所有品牌名称（包括中文名和英文名）
2. 【型号词】提取所有产品型号（如 ATX830, 公爵600, 传奇500）
3. 【品类词】提取所有产品类型叫法（包括同义词、俗称、简称）
   例："山地车" = "山地自行车" = "越野自行车" → 归为一组，标注最常见的叫法
4. 【参数词】提取所有规格参数组合（尺寸+类型，如"26寸山地"、"27速油碟"）
5. 【新兴词】提取最近出现或罕见的品类词/叫法（你可能不认识的词）
6. 【价格区间词】提取含价格信号的词（如"千元山地车"、"入门公路车"）

约束：
- 不要提取卖家状态的词（如"闲置自用""搬家便宜出""仅自提""不议价"）
- 不要提取非品类相关的形容词（如"九成新""自用""很新"）
- 同一个东西的不同叫法合并为一组，选最常见的作为代表词
- 对于新兴词，标注你的确认度（高/中/低）

标题列表：
{titles}
```

## 第三步：AI 三层筛选

候选词进池之前，过三层筛选：

### 层一：去噪

把明显不该进入采集队列的候选词标记为噪声。

Prompt 给 AI：
```
对于以下候选搜索词，判断是否为"噪声词"。

噪声词定义：
- 卖家状态描述（闲置、自用、搬家、清仓、转让）
- 非品类形容词（九成新、99新、几乎全新、正品保证）
- 不含任何商品信息的词（"出""卖""处理"）
- 非交易类信息（"面交""仅自提""可刀"）

如果为噪声词，返回 false；如果可能是有效搜索词，返回 true 并给出理由。

候选词列表：
{candidates}
```

### 层二：去重 + 语义聚合

```
对于以下通过去噪的候选词：

1. 把相同商品/品类的不同叫法合并为一个代表词
   例："山地车""山地自行车""MTB" → 代表词："山地车"
   
2. 判断两个候选词搜出来的商品重叠度是否 > 70%
   如果 > 70%，合并为一个词（选更精准/更通用的那个）

3. 输出去重后的词表，标注每个词的"信息增量"（高/中/低）
   信息增量 = 用这个词搜出来的商品，有多少是用已有词搜不到的
```

这一步关键：**避免三个近义词搜出同一批商品，浪费采集资源。**

### 层三：价值评分

对每个通过去重的候选词打分，决定是否进入下轮筛选。

```
你是一个闲鱼选品策略分析师。请对以下候选搜索词进行价值评估。

每个词根据以下维度评分（0-5分）：

1. 【精准度】：这个词搜出来的结果有多大概率是同一个商品？越高越好。
   5 = 精确到型号/规格（"捷安特ATX830 27速"）
   1 = 宽泛品类（"自行车"）

2. 【需求信号】：从标题中反推这个词对应的商品有人买吗？标题中"已售100+"多吗？
   5 = 多个标题显示高已售
   1 = 没有已售信号

3. 【竞争预估】：标题数量多 vs 已售信号的比例。标题少但已售多 = 蓝海。
   5 = 极低竞争（<5个卖家，已售却很高）
   1 = 饱和（几十个卖家在卖，价格战）

4. 【货源可得性】：拼多多/1688上能找到同款货源的预估概率。
   5 = 明确品牌型号，容易找同款
   1 = 孤品/手作/无品牌，找不到同款

5. 【利润空间】：这个词对应的商品，闲鱼价和拼多多价的预估差价。
   5 = 高利润（如品牌自行车，差价大）
   1 = 低利润（如手机壳，近乎透明）

综合分 = 精准度×1.5 + 需求信号×1.5 + 竞争预估×2.0 + 货源可得性×1.5 + 利润空间×1.5

综合分 ≥ 6.0 → 进入下轮搜索
综合分 4.0-5.9 → 标记为"待观察"，暂不进队列
综合分 < 4.0 → 淘汰

候选词列表：
{candidates}

输出 JSON 格式：
{
  "passed": [{"word": "...", "score": 7.5, "reason": "..."}],
  "watch":  [...],
  "discarded": [...]
}
```

## 第四步：入队 + 下一轮

通过三层筛选的词进入采集队列，作为下一轮飞轮的搜索词。同时更新词库状态：

```
词库状态追踪：

┌───────┬──────────┬────────┬──────┬──────────┬────────────┐
│ 词     │ 评分     │ 来源轮次│ 已采集│ 产出标题数 │ 状态        │
├───────┼──────────┼────────┼──────┼──────────┼────────────┤
│ 自行车  │ S/85     │ 种子    │ ✅   │ 60条      │ 已完成      │
│ 瓜车    │ A/7.5    │ 第1轮   │ ⏳   │ -         │ 待采集      │
│ 死飞    │ B/4.5    │ 第1轮   │ -    │ -         │ 待观察      │
│ 闲置自用│ D/0      │ 第1轮   │ -    │ -         │ 已淘汰(噪声) │
└───────┴──────────┴────────┴──────┴──────────┴────────────┘
```

## 飞轮的控制机制

### 1. 每轮上限

每轮最多让 N 个新词进入采集队列。比如：
- 第1轮：种子词 10 个
- 第2轮：通过筛选的一级候选词最多 30 个
- 第3轮：二级候选词最多 50 个

避免词库爆炸。

### 2. 品类多样性保护

每轮强制保留至少 3 个**非主流品类**的新兴词（即使评分不够高），避免偏见循环。

```
规则：如果本轮通过筛选的候选词中，某品类（如"数码"）占比 > 40%，
则从其他品类的"待观察"词中提权保留 1-2 个。
```

### 3. 词过期淘汰

```python
# 如果某词进入词库30天后仍未产生任何S/A级商品，标记为"过期"
if word.added_days_ago > 30 and word.produced_good_products == 0:
    word.status = "expired"
```

### 4. 人工闸门

AI 筛选后的结果不是自动执行，而是展示给用户确认。用户可以从 "passed" 中踢掉不想要的，从 "watch" 中手动提权。

## 飞轮跑起来的效果模拟

```
种子：10个手工词（自行车、电饭煲、机械键盘、床、台灯...）

第1轮飞轮：
  搜索10个种子词 → 收200条标题 → AI提取80个候选 → 筛选通过25个
  新词库：35个（10种子 + 25新）

第2轮飞轮：
  搜索25个A+词 → 收500条标题 → AI提取150个候选 → 筛选通过40个
  新词库：60个（淘汰15个低效种子词后）

第3轮飞轮：
  搜索40个A+词 → 收800条标题 → AI提取200个候选 → 筛选通过50个
  新词库：85个

三轮过后：10个手工词 → 85个AI筛选过的精准词
其中包含你不可能手动发现的词：瓜车、固齿、苏泊尔SF40FC873、Cherry MX3.0S...
```

## AI 选哪个

| 环节 | 推荐模型 | 原因 |
|------|---------|------|
| 提取候选词 | DeepSeek | 长文本结构化提取能力强，成本低 |
| 去噪筛选 | 智谱 GLM-4 | 简单的二分类任务，速度快 |
| 价值评分 | DeepSeek | 需要多维度和品类知识 |

或者都统一用 DeepSeek（简单省事），成本估算：每轮飞轮约 3-5 次 API 调用，总 token 量约 10K-20K，费用约 ¥0.1-0.3。
