## 一、去噪：两道防线

### 第一道：规则前置（免费，毫秒级）

在调 AI 之前，先用正则把明显垃圾干掉。这部分不花钱。

```python
NOISE_PATTERNS = [
    # 卖家状态
    r"闲置|自用|搬家|清仓|转让|处理|出手|不要了|不用了|换钱",
    # 成色描述（单独出现，不附带品牌型号时才是噪声）
    r"^(九成新|九九新|99新|几乎全新|全新未拆|八成新|七成新)$",
    # 交易方式
    r"面交|自提|仅自提|可刀|不议价|不刀|包邮|到付|同城",
    # 无信息量的单字/短语
    r"^(出|卖|售|转|送|卖掉了|已出|已卖)$",
    # 纯情感/无商品信息
    r"正品保证|支持验货|假一赔十|诚信交易",
    # 非商品链接描述
    r"看我主页|看描述|看图|私聊|咨询|加微信",
]

def rule_filter(word: str) -> bool:
    """返回 True = 保留, False = 噪声淘汰"""
    # 太短 = 大概率无意义
    if len(word) <= 1:
        return False
    # 纯数字或纯符号
    if word.isdigit() or not any(c.isalpha() for c in word):
        return False
    # 匹配噪声模式
    for pattern in NOISE_PATTERNS:
        if re.search(pattern, word):
            return False
    return True
```

规则能干掉约 60-70% 的噪声。剩下的交给 AI。

### 第二道：AI 语义去噪

规则处理后留下的词，有些规则判断不了：

```
规则通过但实际是噪声：
  "学生党自用机" → 规则没命中"学生党"，但这不是品类词
  "退坑回血"     → 规则没命中，但这是卖家动机描述
  "年会奖品"     → 同上
  "前男友送的"   → 同上
```

这些需要 AI 做语义判断。

```
Prompt：
┌──────────────────────────────────────────────────────┐
│ 你的任务：判断以下词语能否作为闲鱼选品的有效搜索词。      │
│                                                      │
│ 有效搜索词 = 描述一个具体的、可交易的商品或品类          │
│                                                      │
│ 无效（噪声）= 描述卖家状态、卖家动机、交易条件、         │
│              情感表达、成色评价、渠道来源               │
│                                                      │
│ 疑似噪声但可能有效的情况（标为 watch，不放行）：         │
│  - "年会奖品"可能是品类（年会奖品一般是什么品类？不确定） │
│  - "退坑"在某些品类是正常叫法（手办退坑、钓鱼退坑）       │
│                                                      │
│ 对每个词回答：有效/噪声/疑似                            │
│ 噪声需要说明理由                                       │
│ 疑似需要说明"可能属于哪个品类"                          │
│                                                      │
│ 词语列表：                                            │
│ {words}                                              │
└──────────────────────────────────────────────────────┘
```

### 去噪总结

```
原始候选词（150个）
    │
    ▼ 规则前置（免费）→ 干掉 ~90 个明显垃圾
    │
60个进入AI
    │
    ▼ AI语义去噪 → 干掉 ~20 个（动机描述、情感表达等）
    │              标记 ~5 个为"疑似"
    │
35个干净候选词 → 进入去重+评分
```

## 二、AI 评分：五个维度 + 综合分

### 评分 Prompt

```
你是一个闲鱼选品策略分析师。现在给你一批"候选搜索词"，请对每个词按以下5个维度评分（0-5分整数）。这些词将用于在闲鱼上搜索商品，目标是找到"能在拼多多/1688找到货源、且在闲鱼有利润空间"的商品。

═══════════════════════════════════════════════════
维度1：【精准度】(0-5) —— 搜出来的结果会是同一种商品吗？
═══════════════════════════════════════════════════

5 = 精确到具体型号/规格，搜索结果高度一致
    例："捷安特ATX830 27速" → 搜出来都是同一款车
    例："Cherry MX3.0S 红轴" → 结果是同一个键盘型号

3 = 品类+关键参数，搜索结果属于同一品类但型号混杂
    例："铝合金山地车 27速" → 都是27速山地车但品牌型号各异
    例："4L电饭煲 智能" → 都是同类产品但不全是同一款

1 = 宽泛品类词，搜索结果五花八门
    例："自行车" → 童车、山地、公路、配件全混在一起
    例："手机壳" → iPhone壳、华为壳、透明壳、硅胶壳全有

───────────────────────────────────────────────────
维度2：【需求信号】(0-5) —— 从标题特征判断这个词对应的商品在闲鱼上有人买
───────────────────────────────────────────────────

判断依据（从提供的标题中分析）：
- 标题中是否出现"已售100+""已售1000+"等标签？
- "想要"数是否普遍高？
- 是否有较多卖家在用这个词？

注意：你是从一批已有标题推测需求，不是精确测量。
如果对某个词完全没有证据，打3分（中性），不要打0。

5 = 多个标题显示高已售/高想要，需求信号强
3 = 有几个标题有中等需求信号
1 = 几乎无需求信号，看起来是冷门词

───────────────────────────────────────────────────
维度3：【竞争预估】(0-5) —— 闲鱼上卖这个词对应商品的卖家多吗？
───────────────────────────────────────────────────

5 = 极低竞争。标题数少但每条都有高已售，蓝海信号
    例：某个特定型号的相机电池，只有2个人在卖但"已售100+"
4 = 低竞争。品牌型号精确，卖家有限
3 = 中等。常见品类，几十个卖家在售
2 = 偏高。热门品类，上百卖家，价格战迹象
1 = 饱和。几百上千卖家，价格高度透明，无差异化空间

───────────────────────────────────────────────────
维度4：【货源可得性】(0-5) —— 拼多多/1688上能找到同款的概率
───────────────────────────────────────────────────

5 = 明确品牌型号，标品，拼多多一定有货
    例："美的电饭煲 MB-FB40E503" → 品牌+型号明确
    例："华为FreeBuds Pro 3" → 标品，供应链成熟

3 = 品类明确但无精确型号，能找到同品类但不保证同款
    例："实木床 1.8米" → 品类明确但非标，需要进一步确认

1 = 非标品/孤品/手作/无品牌，几乎不可能找到同款
    例："手绘陶瓷杯""古着连衣裙""自编手链"

───────────────────────────────────────────────────
维度5：【利润空间】(0-5) —— 闲鱼价和进货价的预估差距
───────────────────────────────────────────────────

5 = 高利润品类（品牌溢价 + 二手折价差大）
    例：品牌自行车、高端耳机、相机镜头

3 = 中等利润（有一定品牌溢价但价格透明）
    例：品牌小家电、品牌键盘

1 = 低利润/透明（拼多多价和闲鱼价接近）
    例：手机壳、数据线、普通T恤

═══════════════════════════════════════════════════
综合分计算（不用你算，我算）：
═══════════════════════════════════════════════════

composite = 精准度×1.5 + 需求信号×1.5 + 竞争预估×2.0 + 货源可得性×1.5 + 利润空间×1.5

竞争预估权重最高，因为这个飞轮的核心目标就是发现蓝海。

═══════════════════════════════════════════════════
要评分的候选词列表：
═══════════════════════════════════════════════════

{candidates}

═══════════════════════════════════════════════════
输出格式（严格 JSON）：
═══════════════════════════════════════════════════

{
  "scores": [
    {
      "word": "词文本",
      "precision": 4,
      "demand_signal": 3,
      "competition": 5,
      "supply_availability": 4,
      "profit_potential": 3,
      "composite": 0,  // 我算，你填0
      "brief_reason": "品牌型号精确，卖家少且已售高，蓝海特征明显"
    }
  ]
}
```

### 评分后的分类

```python
def classify(composite_score: float) -> str:
    if composite_score >= 7.0:
        return "pass"      # 进入下轮采集
    elif composite_score >= 5.0:
        return "watch"     # 待观察，不采集但保留
    else:
        return "discard"   # 淘汰
```

### 为什么竞争预估权重最高

因为这个飞轮的**核心价值不是找热门词，而是找蓝海词**。一个精准度高、需求强但竞争饱和的词（如"手机壳"），对选品没有意义——货源容易找但利润为零。

反过来，一个精准度高、竞争极低、货源可得性中等但利润高的词（如某个特定型号的自行车），才是最优解。

## 三、控制机制

### 机制一：每轮硬上限

```python
MAX_NEW_WORDS_PER_ROUND = 30  # 每轮最多30个新词入队

def enqueue(candidates: list, current_queue_size: int):
    """通过评分的新词入队，但不超过上限"""
    passed = [c for c in candidates if c["composite"] >= 7.0]
    passed.sort(key=lambda x: x["composite"], reverse=True)
    
    # 如果通过太多，只取 Top-N
    available_slots = MAX_NEW_WORDS_PER_ROUND - current_queue_size
    return passed[:available_slots]
```

### 机制二：品类多样性保护

每轮检查品类分布，强制保留少数品类的新词：

```python
MIN_DIVERSE_CATEGORIES = 3  # 每轮至少3个非主流品类

CATEGORY_MAP = {
    # 通过品牌/关键词自动归类
    "捷安特|美利达|喜德盛|ATX|山地|公路": "自行车",
    "华为|小米|OPPO|vivo|手机壳|充电器": "数码配件",  
    "美的|苏泊尔|九阳|电饭煲|空气炸锅": "小家电",
    # ... 更多映射
}

def ensure_diversity(passed_words: list, watch_words: list) -> list:
    """如果某个品类占比 > 40%，从 watch 中提权其他品类的词"""
    categories = Counter(classify_category(w) for w in passed_words)
    dominant = categories.most_common(1)[0]
    
    if dominant[1] / len(passed_words) > 0.4:
        # 从 watch 中找其他品类的词，提权保留
        other_category_words = [
            w for w in watch_words 
            if classify_category(w) != dominant[0] 
            and w["composite"] >= 5.5  # 接近 pass 线
        ]
        return passed_words + other_category_words[:2]
    
    return passed_words
```

### 机制四：飞轮方向检测

每轮之后检测飞轮是否在退化：

```python
def check_degradation(current_round_words: list, previous_round_words: list) -> dict:
    """检测飞轮是否在退化"""
    # 指标1：平均评分是否下降
    prev_avg = mean(w["composite"] for w in previous_round_words)
    curr_avg = mean(w["composite"] for w in current_round_words)
    
    # 指标2：通过率是否暴跌
    prev_pass_rate = len([w for w in previous_round_words if w["composite"] >= 7.0]) / len(previous_round_words)
    curr_pass_rate = len([w for w in current_round_words if w["composite"] >= 7.0]) / len(current_round_words)
    
    # 指标3：品类集中度
    curr_concentration = max(Counter(classify_category(w) for w in current_round_words).values()) / len(current_round_words)
    
    warnings = []
    if curr_avg < prev_avg - 1.0:
        warnings.append(f"评分均值下降 {prev_avg - curr_avg:.1f}，新词质量在降低")
    if curr_pass_rate < prev_pass_rate * 0.5:
        warnings.append("通过率腰斩，词库可能已接近饱和")
    if curr_concentration > 0.6:
        warnings.append(f"品类集中度 {curr_concentration:.0%}，偏见循环风险")
    
    return {"healthy": len(warnings) == 0, "warnings": warnings}
```

### 机制五：词生命周期管理

```python
WORD_STATES = {
    "seed":      "种子词（手工添加）",
    "active":    "活跃中，已采集且产生过商品",
    "queued":    "通过筛选但尚未采集",
    "watch":     "待观察，已产生少量商品但未达标",
    "expired":   "过期（30天未产生任何S/A商品）",
    "discarded": "已淘汰（评分低或标记为噪声）",
}

def lifecycle_check(word_registry: dict):
    """定期检查词库，更新过期词"""
    now = datetime.now()
    for word, info in word_registry.items():
        days_since_added = (now - info["added_at"]).days
        
        if info["state"] == "active":
            # 30天无产出的活跃词 → 降级为过期
            if days_since_added > 30 and info["produced_goods"] == 0:
                info["state"] = "expired"
                
            # 但如果是品类词（宽泛词），过期时间更长
            if info["is_broad_category"] and days_since_added < 90:
                info["state"] = "active"  # 品类词多给一些时间
    
    return word_registry
```

## 四、完整一轮飞轮的执行流程

```
┌─ 输入：上一轮保留下来的活跃词 (35个)
│
├─ Step 1: 用每个词在闲鱼搜索，收集标题
│   35词 × 20条/词 = 700条标题
│   耗时：约 10-15 分钟（用现有的 V3 采集流水线）
│   费用：0（现有能力）
│
├─ Step 2: 从标题中提取候选词
│   规则提取（品牌正则 + 型号正则 + 参数正则） → 约 200 个
│   AI 提取（DeepSeek 结构化提取）→ 约 80-120 个
│   token 量：~15K，费用：~¥0.1
│
├─ Step 3: 去噪
│   规则去噪 → 过滤掉 ~60% 的噪声 → 剩约 80 个
│   AI 语义去噪 → 过滤掉卖家动机/情感描述 → 剩约 50 个
│   token 量：~8K，费用：~¥0.05
│
├─ Step 4: 去重 + 语义聚合
│   AI 把"山地车/山地自行车/MTB"合并 → 剩约 40 个独立词
│   token 量：~10K，费用：~¥0.07
│
├─ Step 5: 五维评分
│   AI 对每个独立词打分 → 40 个词的评分结果
│   token 量：~12K，费用：~¥0.08
│
├─ Step 6: 分类 + 控制
│   composite ≥ 7.0 → pass（预设 25 个）
│   composite 5.0-6.9 → watch（约 10 个）
│   composite < 5.0 → discard（约 5 个）
│   品类多样性检查 → 从 watch 提权 2 个
│   硬上限 → pass 最多 30 个
│
├─ Step 7: 人工确认
│   展示 pass/watch/discard 清单
│   用户确认/调整
│
└─ Step 8: 入队
    通过确认的词进入采集队列
    更新词库状态
    保存本轮飞轮记录
    
总 token 量：~45K
总 API 费用：~¥0.3（DeepSeek）
总耗时：采集 10-15 分钟 + AI 处理 1-2 分钟 + 人工确认 2-5 分钟
```

## 五、什么情况下飞轮该停

```python
STOP_CONDITIONS = [
    "连续2轮通过率 < 20% → 词库已接近饱和，暂停飞轮",
    "连续2轮平均评分下降 > 1.5 → 新词质量衰减，暂停并人工复盘",
    "品类集中度连续2轮 > 60% → 偏见循环，需要人工注入新品类种子词",
    "待采集队列 > 100个 → 先消化现有队列再开新轮",
    "本轮通过的新词中 > 30% 已在词库中存在 → 语义去重不足，优化 AI prompt",
]
```
