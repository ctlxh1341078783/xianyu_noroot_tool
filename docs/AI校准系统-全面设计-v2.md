# 第一部分：闲鱼搜索词飞轮（角色③）

## 一、每轮需要搜多少页

这个问题直接决定飞轮的成本和可靠性。

### 不是搜多少页的问题，是数据用途不同，页数不同

```
父级搜索（已有数据，免费复用）          阶段C轻量搜索（新增，有成本）
─────────────────────────────          ─────────────────────────
来源：上一轮采集或上一轮飞轮              来源：专门为pending词搜索
页数：看上一轮采集时设了多少页             页数：1页（固定）
用途：阶段B分析信号是否存在               用途：验证阶段B给不出的信号
成本：0（已有数据）                       成本：1次搜索/词，加风控间隔
```

### 父级搜索：最少1页，理想3页

```
1页（20条）                       3页（60条）
─────────────────────────────     ─────────────────────────────
优点：轻量，每个父级词1次搜索       优点：样本量翻3倍，信号更可靠
缺点：样本小，候选词可能没出现      缺点：搜索次数×3，风控成本×3
适用：父级词本身是精准词            适用：父级词是宽泛品类词
      "捷安特ATX830"搜1页够              "自行车"搜3页才够覆盖子品类
```

**建议策略：首次搜索宽泛词用3页，精准词用1页。后续飞轮只看这3页的数据，不额外搜。**

```python
def determine_search_pages(word_type: str, is_first_round: bool) -> int:
    """
    宽泛品类词（"自行车"）→ 3页，因为需要覆盖更多子品类才能给阶段B足够信号
    精准词（"捷安特ATX830"）→ 1页，结果已经很集中
    飞轮后续轮次 → 复用已有数据，不额外搜
    """
    if not is_first_round:
        return 0  # 复用已有，不搜
    
    if word_type == "broad_category":
        return 3
    elif word_type == "brand_model":
        return 1
    else:
        return 1
```

### 阶段C：固定1页

阶段C的目的不是完整评估一个词，而是**验证信号是否存在**。1页就够了：
- `numFound` → 竞争规模一目了然
- 20条商品 → 售出率、均价、卖家结构都可以算
- 不需要翻页，因为不需要完整行情数据

如果1页后仍无法判断 → 这个词不应该进词库，直接标记 watch。

### 样本够不够的判断

```
候选词在父级结果中出现 ≥ 3次  → 样本够，可以打分
候选词在父级结果中出现 1-2次  → 样本偏少，保守打分（上限4分）
候选词在父级结果中出现 0次    → 样本不够，标记 pending_verify
```

如果父级搜索只有1页（20条），很多候选词可能只出现0-1次，导致大量进入pending_verify。**这就是为什么要宽泛品类词搜3页**——放大父级搜索的覆盖范围，让阶段B能直接判断更多候选词，减少阶段C的搜索次数。

## 三、阶段C —— 轻量搜索验证

### 执行条件

阶段B标记为 `pending_verify` 的词，且满足：词结构有"品牌+型号"或"品类+精确参数"特征（值得验证的）。

### 搜索页数

**固定1页**。原因：
- 阶段C的目的是验证信号是否存在，不是完整评估
- `numFound` + 20条的售出率/均价/卖家结构，足够判断
- 每词多搜1页，成本翻倍，但信息增量有限（20条已经能看出模式）

如果1页后仍判断不了 → 这个词信号太弱，标记 watch，不进词库。

### 执行

```python
def phase_c_verify(pending_words: list, bridge: FridaBridge) -> dict:
    results = {}
    
    for i, word_info in enumerate(pending_words):
        # 风控间隔
        delay = random.uniform(15, 25)
        time.sleep(delay)
        
        # 搜索1页
        search_result = bridge.search(word_info["word"], page=1, page_size=20)
        
        results[word_info["word"]] = {
            "numFound": search_result.get("numFound", 0),
            "summary": {
                "total_items": 20,
                "items_with_soldCount": count_sold(search_result),
                "soldCount_distribution": sold_dist(search_result),
                "avg_price": avg_price(search_result),
                "avg_wantNum": avg_want(search_result),
                "professional_seller_ratio": prof_ratio(search_result),
                "top_service_tags": top_tags(search_result)
            }
        }
    
    return results
```

# 第二部分：PDD 搜索词优化（角色③扩展）

## 问题

```
现在：闲鱼标题 → 直接搜索拼多多
问题：闲鱼标题给买家看的，不是给拼多多搜索引擎用的
```

## 方案

```
闲鱼商品标题
      │
      ▼
  AI 提取核心产品词 → 生成4种策略的PDD变体
      │
      ▼
  按优先级搜索拼多多 → AI评估匹配质量
      │
      ▼
  品类搜索知识库（下次同类商品直接用最优策略）
```

### 四种搜索策略

```
策略A 精准匹配：品牌+型号          策略B 品类规格：品类+参数+尺寸
策略C 功能属性：功能描述词          策略D 泛品类：最宽泛的品类词
```

### 学习循环

```
第一次搜"自行车"类 → 试A/B/C/D → A策略效果最好 → 写入知识库
第二次搜"自行车"类 → 知识库说A优先 → 先试A → 命中跳过B/C/D → 命中不中再试B
第十次 → A策略命中率8/10 → 品类规则确认
```

# 第三部分：角色② — 品类评分进化

## 流程

```
每条流水线完成 → 写入预测日志（评分 vs 实际利润）
        │
        ▼
单品类积累 ≥ 5条记录 → 触发AI分析
        │
        ▼
AI分析各维度与实际利润的相关性 → 提出权重调整
        │
        ▼
历史样本回测 → 新排序比旧排序好？ → 更新 models_config.json
```

## AI分析要点

```
1. 哪个维度评分和实际利润最一致？最不一致？
2. 品类特有偏差：自行车品类的 supply_attribute 是否无关？
3. 飞轮词 vs 手工词的表现对比
4. 保守调整（≤30%），必须≥3条样本支持
```

# 第四部分：控制机制总览

| 机制 | 检测 | 触发 | 动作 |
|------|------|------|------|
| 品类重叠 | 阶段B AI语义去重 | 重叠>70% | 合并为1个代表词 |
| 偏见循环 | 品类集中度趋势 | 主导>50%且增长 | 提权流失品类词+提主导品类阈值 |
| 通过率异常 | pass率 | <5%或>60% | AI诊断+自动暂停 |
| 队列上限 | 待采集词计数 | >80 | 新词降级watch |
| 边际衰减 | 新词vs词库重叠率 | >60% | 暂停飞轮 |
| 自动降级 | 实际产出跟踪 | 30天无S/A产出 | 过期淘汰 |
| 风控熔断 | 搜索间隔 | <12秒/次 | 拉长间隔+暂停 |
