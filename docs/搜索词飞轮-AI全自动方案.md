# 搜索词飞轮 — AI 全自动方案

## 核心改动

1. **评分维度重做**：不依赖采集数据，AI 只看词本身 + 品类常识就能判断
2. **噪声词不浪费**：按类别存入素材库，上架时自动推荐加入标题
3. **全流程 AI 自动化**：去掉人工闸门，用保守阈值 + 自动降级 + 异常检测代替

---

## 一、重新设计评分维度（采集前判断）

### 五个维度（0-5分）

```
═══════════════════════════════════════════════════
维度1：商品指向性 (0-5) — 这个词能定位到多具体的东西？
═══════════════════════════════════════════════════

不需要采集数据。AI 看词的结构：

5分 = 品牌 + 型号 + 参数，精确到单品
     "捷安特ATX830 27速铝合金" → 一个SKU
     "美的MB-FB40E503 4L电饭煲" → 一个SKU

4分 = 品牌 + 型号，或品类 + 精确参数
     "捷安特ATX830" → 一个型号
     "27.5寸气压前叉山地车" → 精确参数组合

3分 = 品牌 + 品类，或品类 + 一个参数
     "捷安特山地车" → 品牌+品类
     "铝合金车架自行车" → 品类+材质

2分 = 品类词较明确，有细分属性但无品牌
     "山地车 27速" → 品类+参数但无品牌
     "智能电饭煲 4L" → 品类+规格

1分 = 宽泛品类词，无任何细分
     "自行车""电饭煲""键盘"

0分 = 无法定位到任何具体商品
     "好物""推荐""必买"
───────────────────────────────────────────────────
维度2：交易价值 (0-5) — 这种东西在闲鱼上会被买卖吗？
───────────────────────────────────────────────────

AI 凭品类常识判断，不需要数据：

5分 = 高频交易的标品，二手市场成熟
     → 手机、相机、游戏机、品牌自行车、大牌家电
     
4分 = 常见二手品类，有一定交易量  
     → 小家电、键盘鼠标、家具、母婴用品

3分 = 可交易但非主流品类
     → 书籍、文具、园艺工具、宠物用品

2分 = 交易频次低，或二手需求弱
     → 特定专业工具、特殊尺寸配件、冷门收藏

1分 = 几乎不在闲鱼交易
     → 服务、课程、虚拟商品、食品

0分 = 完全不适合闲鱼
     → 活体、违禁品、已拆封化妆品
───────────────────────────────────────────────────
维度3：货源可得性 (0-5) — 拼多多/1688能找到货吗？
───────────────────────────────────────────────────

AI 凭供应链常识判断：

5分 = 品牌标品，大货充足，拼多多一定有
     → 品牌家电、品牌数码、品牌自行车配件
     → 判断标志：有明确品牌+型号，属于制造业量产范畴

4分 = 有品牌但在拼多多上可能是经销商/分销商出
     → 品牌服装、品牌鞋履、品牌箱包
     → 判断标志：有品牌但可能不是品牌直营

3分 = 无品牌但有明确品类，能找到同品类不同品牌
     → 实木床1.8米、升降桌、人体工学椅
     → 判断标志：品类成熟、工厂多，但无具体品牌锚定

2分 = 能找到近似品，但同款概率低
     → 复古台灯、日式餐具、手工皮具
     → 判断标志：半标品，有规格但非标准工业品

1分 = 孤品/手作/古董/二手特供，几乎无货源
     → 手绘陶瓷、古着、孤品家具、老物件
     → 判断标志：非标品，每件商品独一无二

0分 = 不可复制，无货源概念
     → 个人服务、二手课本笔记、私人定制
───────────────────────────────────────────────────
维度4：利润结构 (0-5) — 闲鱼价和进货价的典型关系？
───────────────────────────────────────────────────

AI 凭品类常识判断（不是精确计算，而是定性判断）：

5分 = 高溢价品类
     品牌溢价 + 二手折价差大 + 信息不对称
     → 品牌自行车（进货3000卖4500）、相机镜头、高端耳机
     
4分 = 中等溢价
     有品牌溢价但价格较透明
     → 品牌小家电、机械键盘、品牌灯具

3分 = 一般利润
     品类成熟、价格透明，差价在10-25%
     → 普通家电、日用百货、基础家具

2分 = 低利润
     接近透明市场，差价 < 10%
     → 手机配件、数据线、普通手机壳

1分 = 无利润空间
     闲鱼价 ≤ 拼多多新品价（二手比新的还便宜或持平）
     → 部分快消品、部分日用品

0分 = 无法变现
     → 服务类、虚拟类
───────────────────────────────────────────────────
维度5：词库边际贡献 (0-5) — 这个词能给词库带来多少新商品？
───────────────────────────────────────────────────

AI 跟已有词库比对，判断增量价值：

5分 = 完全新品类/新细分，词库里没有任何近似词
     → 词库有"自行车"但没有"瓜车/gravel"

4分 = 已有品类的细分，但足够不同
     → 词库有"电饭煲"但没有"迷你电饭煲"

3分 = 已有品类的另一种叫法，搜出来和已有词有50%不同
     → 词库有"山地车"，新增"山地自行车"→ 有增量但不全是

2分 = 近义词，搜索结果重叠度 > 70%
     → 词库有"机械键盘"，新增"机械键盘 青轴"→ 重叠高但覆盖子集

1分 = 几乎完全重叠
     → 词库有"自行车 26寸"，新增"26寸自行车"→ 只是词序不同

0分 = 完全重复
     → 词库已有完全相同或仅差1个字符的词

═══════════════════════════════════════════════════
综合分（AI 不需要算，代码算）：
═══════════════════════════════════════════════════

composite = 商品指向性×2.0 + 交易价值×1.5 + 货源可得性×2.0 + 利润结构×1.5 + 词库边际贡献×2.0

解释：
- 商品指向性×2.0：飞轮的核心目标是精准词
- 货源可得性×2.0：找不到货源，其他维度全是空的
- 词库边际贡献×2.0：近义词没有任何增量价值
- 交易价值×1.5：闲鱼本身就有交易才值得采
- 利润结构×1.5：有利润才有动机

满分 = 5×(2.0+1.5+2.0+1.5+2.0) = 5×9.0 = 45
归一化到0-10：composite_10 = composite / 45 × 10
```

## 三、全 AI 自动化流程

### 整体架构

```
一轮飞轮 = 一次 AI 调用，同时完成：分词提取 + 分流 + 去重 + 评分 + 素材归类

┌─────────────────────────────────────────────────────────┐
│  输入：上轮保留的活跃词 (N个) + 已有词库 + 已有素材库      │
│                                                         │
│  用这N个词在闲鱼搜索 → 收集标题 → 从标题提取候选词串      │
│                                                         │
│  ┌─────────────────────────────────────────────────┐    │
│  │       一次 AI 调用，输出完整 JSON               │    │
│  │                                                 │    │
│  │  search_words[]  → 已评分+已去重+已分类         │    │
│  │  title_materials{} → 按场景归类的标题素材        │    │
│  │  noise[]          → 淘汰词+理由                 │    │
│  │  notes            → AI 的观察和建议              │    │
│  └─────────────────────────────────────────────────┘    │
│                         │                               │
│                         ▼                               │
│  ┌─────────────────────────────────────────────────┐    │
│  │              自动分流                            │    │
│  │                                                 │    │
│  │  composite_10 ≥ 7.5 → 入队（下轮采集词）         │    │
│  │  composite_10 6.0-7.4 → 待观察池                 │    │
│  │  composite_10 < 6.0 → 淘汰                       │    │
│  │  title_boost → 自动合并到素材库                   │    │
│  └─────────────────────────────────────────────────┘    │
│                                                         │
│  输出：更新后的词库 + 素材库 + 飞轮日志                   │
└─────────────────────────────────────────────────────────┘
```

### 自动控制机制（替代人工闸门）

```python
class AutoGuard:
    """全自动控制，替代人工闸门"""
    
    PASS_THRESHOLD = 7.5      # 比之前更保守（原7.0），自动模式要更严格
    WATCH_THRESHOLD = 6.0
    MAX_NEW_PER_ROUND = 30
    MAX_QUEUE_SIZE = 80       # 待采集队列上限
    
    def auto_execute(self, ai_output: dict, state: FlywheelState):
        """AI输出 → 自动决策 → 更新状态"""
        
        search_words = ai_output["search_words"]
        warnings = []
        actions = []
        
        # ── 1. 分类 ──
        for word in search_words:
            score = word["composite_10"]
            if score >= self.PASS_THRESHOLD:
                word["status"] = "pass"
            elif score >= self.WATCH_THRESHOLD:
                word["status"] = "watch"
            else:
                word["status"] = "discard"
        
        # ── 2. 品类多样性保底 ──
        passed = [w for w in search_words if w["status"] == "pass"]
        if self._concentration_risk(passed):
            rescued = self._rescue_other_categories(search_words, passed)
            actions.append(f"品类多样性保护：从watch提权 {len(rescued)} 个词")
        
        # ── 3. 队列上限 ──
        if state.queue_size + len(passed) > self.MAX_QUEUE_SIZE:
            overflow = state.queue_size + len(passed) - self.MAX_QUEUE_SIZE
            # 只保留最高分的，其余的降级为 watch
            passed.sort(key=lambda x: x["composite_10"], reverse=True)
            downgraded = passed[-overflow:]
            for w in downgraded:
                w["status"] = "watch"
            passed = passed[:-overflow]
            actions.append(f"队列上限：{overflow} 个词降级为 watch")
        
        # ── 4. 异常检测 ──
        pass_rate = len(passed) / len(search_words) if search_words else 0
        
        if pass_rate < 0.1 and state.round_number > 1:
            warnings.append(
                f"⚠️ 本轮通过率仅 {pass_rate:.0%}，可能原因：\n"
                f"  1) 词库已饱和，继续飞轮收益低\n"
                f"  2) 本轮搜索词质量下降\n"
                f"  自动暂停飞轮，建议人工检查后重启"
            )
            state.auto_paused = True
        
        if pass_rate > 0.8 and state.round_number > 1:
            warnings.append(
                f"⚠️ 本轮通过率异常高 ({pass_rate:.0%})，可能评分标准松动\n"
                f"  建议人工抽查 high-pass 词的评分是否合理"
            )
        
        # ── 5. 自动降级检测 ──
        if state.round_number >= 3:
            recent_scores = state.get_recent_round_scores(3)
            if self._trend_down(recent_scores):
                warnings.append(
                    f"⚠️ 连续3轮评分均值下降：{recent_scores}\n"
                    f"  词库新词的边际价值在衰减，建议考虑人工注入新品类种子词"
                )
        
        # ── 6. 标题素材自动合并 ──
        new_materials = ai_output.get("title_materials", {})
        merged_count = state.title_library.merge(new_materials)
        if merged_count > 0:
            actions.append(f"标题素材库新增 {merged_count} 条素材")
        
        # ── 7. 写日志 ──
        self._write_round_log(state.round_number, ai_output, passed, warnings, actions)
        
        return {
            "passed": passed,
            "watch": [w for w in search_words if w["status"] == "watch"],
            "discarded": [w for w in search_words if w["status"] == "discard"],
            "materials_merged": merged_count,
            "warnings": warnings,
            "actions": actions,
            "auto_paused": state.auto_paused,
        }
```

### 不需要人工闸门的底气

| 保护机制 | 作用 |
|---------|------|
| 通过阈值提到 7.5 | 宁可漏过也不放进垃圾词 |
| 队列上限 80 个 | 不爆仓，不会让采集成本失控 |
| 品类多样性保底 | 自动打破偏见循环 |
| 异常通过率检测 | 过高/过低自动报警或暂停 |
| 连续3轮评分衰减 | 自动暂停，等人工注入新品类种子 |
| 全部决策可追溯 | 每轮都有日志，事后可以审计 AI 为什么做了某个决定 |

## 四、完整流程时序

```
第0轮（手动初始化）
  用户输入 10 个种子词 → 直接入队 → 进入第1轮

第1轮（首次飞轮）
  Step 1: 用10个种子词搜索闲鱼 → 收集200条标题
  Step 2: 从标题提取候选词串 → 去标点、去URL、去emoji → ~300个词串
  Step 3: 一次 AI 调用 → 输出 JSON（分流+去重+评分+素材归类）
  Step 4: AutoGuard 自动分流 → 25个词入队，5个watch，15个title_boost素材
  Step 5: 更新词库和素材库
  
第2轮
  Step 1: 用25个A+词搜索闲鱼 → 收集500条标题
  Step 2-5: 同上
  ...

第N轮
  如果 auto_paused = True → 暂停，通知用户检查
  否则继续
```

```
         │ 输出：扩充后的词库 + 标题素材

  手动确认/调整关键词（可选）
         │
         ▼
  行情采集 → 选词评分 → 选品采集 → 货源查找
                                    │
                    上架时 ← 自动匹配标题素材库推荐标题
```

飞轮是一个**独立的前置模块**，跑不跑飞轮都不影响现有流水线。跑了飞轮只是让词库更多、更精准。
